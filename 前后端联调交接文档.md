## 1. 根据后端文档已完成的前端修改

### 1.1 Token 认证机制 ✅

根据后端文档要求，除登录注册外**所有接口必须携带 Token**：

```javascript
// Header 格式
Authorization: Bearer <token>
```

**已修改文件：**
- `js/app.js` - 登录时保存 Token，新增 `getToken()`, `request()` 等方法
- 所有业务页面 - 添加 Token 到请求头

**Token 管理：**
- 登录成功后自动保存到 `plus.storage`
- Token 有效期：**30 分钟**
- 过期后自动跳转登录页

### 1.2 故障管理模块 ✅

**已修改文件：**
- `submit_failure.html` - 新增故障提交
- `failure_details.html` - 故障详情和标记已处理
- `to_be_process.html` - 待处理故障列表

**数据结构对齐：**

| 字段 | 类型 | 说明 | 前端状态 |
|------|------|------|----------|
| `title` | String | 故障标题 | ✅ |
| `content` | String | 故障详情描述 | ✅ |
| `status` | Integer | 1=未处理，0=已处理 | ✅ |
| `lightId` | String | 关联路灯物理编号 | ✅ |
| `lightName` | String | 关联路灯名称 | ✅ 新增 |
| `area` | String | 所属区域 | ✅ |
| `longitude` | BigDecimal | 故障路灯经度 | ✅ 默认0 |
| `latitude` | BigDecimal | 故障路灯纬度 | ✅ 默认0 |
| `createTime` | LocalDateTime | 故障上报时间 | 后端自动 |

**接口变更：**
- 标记已处理：改用 `GET /faults/check?id={id}`

### 1.3 维修人员模块 ✅

**已修改文件：**
- `add_person.html` - 新增维修人员

**数据结构对齐：**

| 字段 | 类型 | 说明 | 前端状态 |
|------|------|------|----------|
| `number` | String | 员工编号（最长18位） | ✅ |

## 2. 后端接口规范摘要

### 2.1 统一响应结构

```json
{
  "code": 200,
  "message": "成功描述",
  "data": {}
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 401 | 未登录 / Token 失效 |
| 403 | 无权限 |
| 404 | 参数验证失败 |
| 500 | 失败 |

### 2.2 重要接口说明

#### 路灯状态控制
```
POST /lamps/stop    - 停止运行 (status=0)
POST /lamps/normal  - 恢复正常 (status=1)
POST /lamps/breaks  - 标记故障 (status=2)

Body: 直接传 Long 类型路灯 id（无包装）
如: 1
```

#### 故障标记已处理
```
GET /faults/check?id={id}
```

#### 手动模式设置角度
```
POST /lamps/setmanual
Content-Type: application/x-www-form-urlencoded

参数: id, threoyHighAngel, threoyDirectionAngel
```
